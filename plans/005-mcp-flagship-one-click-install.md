## Why this matters

The MCP server is the most distributable, most current, and most differentiated
surface in this repo (workflow-first tools, dry-run→confirm-token write safety,
an offline SSRF guard on webhook URLs). There is **no official Clockify MCP** —
the field is fragmented community repos and proxy aggregators — so this is the
strongest entrant. But today the only documented install path is a developer
build (`cd mcp && npm install && npm run build && npm link`, `mcp/README.md:20-27`)
and a non-developer is blocked at step zero. This plan produces the **one-click
install bundle** (a Claude Desktop / MCPB `.mcpb`) and end-user onboarding so the
existing engineering becomes installable in under two minutes. It builds and
verifies the artifact locally but **stops before distribution** (publishing /
GitHub Release), which is a maintainer decision and a repo hard-stop.

  `@modelcontextprotocol/sdk`, `zod`, and the peer `clockify-sdk-ts-115` (the peer
  is imported by 28 compiled files via 13 subpaths). Nothing is bundled/tree-shaken.
  All guide/doc text is inlined as JS constants — **no data assets** to ship beyond
  `dist/**/*.js`.

  pack of `mcp/` ships **no runtime deps** and crashes with `ERR_MODULE_NOT_FOUND`.
  Mitigating fact: the wrapper (`v0.9.0`) has **zero** runtime/peer deps and
  `files:["dist","README.md","LICENSE"]`, so `npm pack -w clockify-sdk-ts-115`
  yields a small, self-contained tarball.

```json
  "license": "MIT",
  "server": {

    "entry_point": "dist/index.js",
    "mcp_config": {
      "command": "node",
      "args": ["${__dirname}/dist/index.js"],
      "env": {
        "CLOCKIFY_API_KEY": "${user_config.api_key}",
        "CLOCKIFY_WORKSPACE_ID": "${user_config.workspace_id}"
      }
    }
  },
  "user_config": {
    "api_key": {
      "type": "string",
      "title": "Clockify API key",
      "description": "Clockify -> Profile -> API. Stored in your OS keychain.",
      "sensitive": true,

      "type": "string",

```

### Step 2: Write `scripts/build-mcpb.mjs` (the staging + pack builder)

`mcpb pack` does NOT install dependencies. Build a self-contained bundle by
staging a real production install. Create `scripts/build-mcpb.mjs` (Node ESM,
match the style of the other `scripts/*.mjs` — `#!/usr/bin/env node`, top-of-file
doc comment, `node:` imports, `process.exit(1)` on failure). It must perform,
using a temp dir under the OS tmpdir (NOT inside the repo):

1. Assert `mcp/dist/index.js` and `wrapper/dist` exist; if not, exit 1 with a
   message telling the operator to build first.
2. `npm pack -w clockify-sdk-ts-115 --pack-destination <tmp>` → capture the
   produced `clockify-sdk-ts-115-<ver>.tgz` path.
3. Create `<tmp>/bundle/`, copy into it: `mcp/dist` → `bundle/dist`, and
   `mcp/manifest.json`, `mcp/README.md`, `mcp/LICENSE` → `bundle/`.
4. Write `<tmp>/bundle/package.json`:
   ```json
   {
     "name": "clockify115-mcp",
     "version": "<mcp version>",
     "type": "module",
     "bin": { "clockify115-mcp": "dist/index.js" },
     "dependencies": {
       "@modelcontextprotocol/sdk": "^1.29.0",
       "zod": "^3.25.0",
       "clockify-sdk-ts-115": "file:<tmp>/clockify-sdk-ts-115-<ver>.tgz"
     }
   }
   ```
   (Read the real `@modelcontextprotocol/sdk` and `zod` ranges from
   `mcp/package.json` rather than hardcoding.)
5. Run, in `<tmp>/bundle`: `npm install --omit=dev --install-links --no-package-lock`
   (`--install-links` forces the `file:` dep to install as a real copy, not a
   symlink).
6. Pack with an explicit output path:
   `npx --yes @anthropic-ai/mcpb@2.1.2 pack <tmp>/bundle <repo>/mcp/clockify115-mcp-<ver>.mcpb`.
7. Print the final `.mcpb` path and exit 0. Leave the temp dir cleanup to a
   `finally` (or print its path so it can be inspected).

**Verify**: `node scripts/build-mcpb.mjs` → exit 0; `ls -la mcp/*.mcpb` shows one
file > 0 bytes; `npx --yes @anthropic-ai/mcpb@2.1.2 info mcp/clockify115-mcp-*.mcpb`
lists `node_modules/@modelcontextprotocol/sdk/`, `node_modules/zod/`,
`node_modules/clockify-sdk-ts-115/`, `dist/index.js`, `manifest.json`.

### Step 3: Self-containment smoke (prove the bundle runs standalone)

Unpack the bundle to an isolated dir with NO access to the monorepo
`node_modules` and confirm it does not crash on module resolution:

```sh
npx --yes @anthropic-ai/mcpb@2.1.2 unpack mcp/clockify115-mcp-*.mcpb /tmp/mcpb-smoke
cd /tmp/mcpb-smoke && CLOCKIFY_API_KEY=dummy CLOCKIFY_WORKSPACE_ID=dummy node dist/index.js
```

Expected: the process does **not** print `ERR_MODULE_NOT_FOUND` / `Cannot find
module`. It may hang (waiting on stdio JSON-RPC) or fail later on auth — both
prove module resolution works. Kill it after ~3s. Do NOT use real credentials.

**Verify**: no `Cannot find module` / `ERR_MODULE_NOT_FOUND` in output.

  (bundle builds + validates + runs standalone) but does not *ship* it. Going
  live = maintainer picks: (a) `npm publish` so `npx @clockify115/mcp-server`
