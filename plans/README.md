## Findings considered and rejected

Recorded so they are not re-audited next pass:

- **Float-dust / NaN-guard / date-bounds / negative-retry-delay nitpicks** (rate-limit.ts, money.ts, dates.ts, composed-fetch.ts): all already safe (fallbacks return undefined; `setTimeout` coerces negatives to 0; money "float dust" is already documented in `money.ts`). Not worth code churn.
- **confirm-token replay / caller-binding hardening** (confirmation.ts): the existing triple barrier (token lookup + argsHash + previewHash, plus 5-min TTL and single-use delete) is adequate; caller-binding is defense-in-depth with no demonstrated gap.
- **GitHub Actions `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24` pin** (ci.yml): by-design, scheduled-maintenance removal once actions default to Node 24; not a defect.
- **"Release requires a manually pushed tag"**: intentional governance (documented in AGENTS.md/CLAUDE.md), not a gap.
- **vitest version skew (wrapper 4.x vs cli/mcp 2.x)** (#11): real but low-leverage dev-only consistency item; the audit-claimed CVE does not apply (wrapper's 4.x is not in the vulnerable <3.2.6 range). Left for a routine dependency-unification pass — not planned now.
- **CLI rc-file plaintext key, no chmod guidance** (#12, config.ts): real but Med-confidence and partly mitigated by env-first precedence; left unplanned pending a maintainer call on whether to warn vs. require a flag.
- **`resolveUserRef` triple-scans the user list** (PERF-001): O(n) at 10–100 users — not a bottleneck.
- **Retry-header parsing duplicated 3×** (TD-001) and **unescaped `{key}` regex in generated path substitution** (GEN-01): real but low-impact (each impl is tested/correct; Clockify path params are alphanumeric so the regex bug can't trigger today). Candidates for a future tech-debt sweep, not selected.
