# AGENTS.md

Canonical contributor + agent contract for this repository
(`apet97/clockify-ts-sdk`). This is a **standalone repo** — it has
no parent project. The `addons-me/` directory you might see in some
absolute paths is one contributor's local workspace folder where
this repo is cloned alongside other unrelated repos; it is not
part of the contract.

The sister repository `apet97/go-clockify` (referred to throughout
this doc as **GOCLMCP**, its conventional local clone directory
name) owns the canonical Clockify OpenAPI generator. We assume the
two repos are cloned as **siblings** so that `../GOCLMCP/...`
paths resolve from this repo's root. Adjust paths if your layout
differs.

Read this whole file before touching anything. The rules below
apply to humans and AI agents equally.

## 1. Identity & boundary

This repository **wraps** the Fern-generated TypeScript SDK for the
Clockify REST API and ships it as the npm package
**`clockify-sdk-ts`**. The publishable artefact is `wrapper/dist/`.
Everything else in the tree is the toolchain that produces it.

What ships on npm:
- `wrapper/dist/**` (built from `wrapper/src/**` via tsc)
- `wrapper/README.md`, `wrapper/LICENSE`, `wrapper/package.json`

What does NOT ship on npm (but lives here for reproducibility):
- `spec/` — Fern config + corrected OpenAPI snapshot + evidence ledger
- `output/ts-sdk/` — raw Fern generator output (regenerable from `spec/`)
- `wrapper/src/`, `wrapper/dist/`, `wrapper/node_modules/` — git-ignored,
  recreated by the build chain
- `.github/workflows/` — CI + release pipelines
- `spec/evidence/probes/*.{json,hdr}` — git-ignored live API captures

The **canonical Clockify OpenAPI spec is NOT in this repo.** It is
generated by `scripts/gen-clockify-openapi` in the sister repo
`apet97/go-clockify` (conventionally cloned next to this repo as
`../GOCLMCP/`) from a curated source bundle
(`docs/openapi/sources/{AIII,realOPENAPI,clockify-api-probe-lab}`).
The file at `spec/corrected/clockify.corrected.openapi.yaml` is a
**snapshot** of that canonical, refreshed by `cp` after every regen
in GOCLMCP.

## 2. First reads (in order)

1. `README.md` (this repo) — high-level workflow overview.
2. `wrapper/README.md` — the user-facing SDK README that ships on npm.
   Reflects how external consumers see the SDK.
3. `spec/evidence/discrepancies.md` — the ledger of every divergence
   between Clockify's published spec, the live API behaviour, and the
   shape we ship. Five-question format per entry: official claim,
   actual behaviour, live evidence, MCP tools affected, open
   questions, status. Read this before adding any new annotation,
   override, or workaround — it almost certainly has prior context.
4. `../GOCLMCP/scripts/gen-clockify-openapi` (sister repo
   `apet97/go-clockify`) — the Ruby generator. Sections to know:
   - `TAG_RENAMES` (collapses singular/plural tag duplicates)
   - `PATH_PARAM_PATTERNS` + `stamp_path_param_patterns!` (stamps
     `^[0-9a-fA-F]{24}$` on `expenseId` / `invoiceId` / `assignmentId`)
   - `PAGINATED_LIST_OPS` + `ensure_pagination!` (stamps `page` +
     `page-size` query params on 18 list endpoints)
   - A `NOTE` block where a `derive_sdk_method_name` heuristic was
     parked after it caused Fern to silently drop 12 resource modules
     (see `discrepancies.md` →
     `fern.x-fern-sdk-method-name.drops-resource-modules`).
5. `spec/fern/{fern.config.json,generators.yml}` — Fern workspace
   config. `organization: clockify` drives the emitted class name
   `ClockifyApiClient`. Generator pinned to
   `fernapi/fern-typescript-node-sdk:3.71.2`; Fern CLI pinned to
   `5.37.9`.

## 3. The build chain (top to bottom)

Each arrow is a *script + invocation* that must succeed for the next
stage to be valid. Never skip a stage; never run the publish step
without all upstream gates green.

```text
upstream sources (GOCLMCP/docs/openapi/sources/**)
        │
        │  (cd ../GOCLMCP && make gen-openapi)
        ▼
GOCLMCP/docs/openapi/clockify-openapi.yaml  (canonical, 193 ops)
        │
        │  make {openapi,catalog,selfinspect,raw-allowlist}-drift   ← all 4 must EXIT 0
        │  go test ./internal/tools/...                              ← must pass
        │
        │  cp ../GOCLMCP/docs/openapi/clockify-openapi.yaml \
        │     spec/corrected/clockify.corrected.openapi.yaml
        ▼
spec/corrected/clockify.corrected.openapi.yaml  (frozen snapshot)
        │
        │  (cd spec/fern && fern check --warnings --from-openapi)   ← All checks passed
        │  (cd spec/fern && fern generate --group ts --local --force)
        ▼
output/ts-sdk/**  (Fern emits ~720 TS files; --force WIPES the tree)
        │
        │  cd wrapper && npm run sync       (rsync into wrapper/src/, skipping
        │                                    Fern's smoke-test scaffold files)
        ▼
wrapper/src/**  (gitignored; populated by sync)
        │
        │  npm run type-check               (tsc --noEmit; covers src/**, index.ts,
        │                                     create-client.ts, composed-fetch.ts,
        │                                     iter.ts, webhooks.ts, pagination.ts,
        │                                     tests/**)
        │  npm test                          (vitest; 8 pagination + 8 createClient
        │                                     + 30 iter + 16 webhooks + 26 composed-fetch
        │                                     unit tests + 5 live sandbox flows; live
        │                                     tests skip without CLOCKIFY_API_KEY +
        │                                     CLOCKIFY_WORKSPACE_ID)
        │  npm run build                     (twin tsc passes:
        │                                     `tsc -p tsconfig.esm.json` → dist/esm/**,
        │                                     `tsc -p tsconfig.cjs.json` → dist/cjs/**,
        │                                     then `scripts/finalize-cjs.sh` writes
        │                                     dist/cjs/package.json { type: commonjs }.
        │                                     Hand-written modules emit flat at
        │                                     dist/{esm,cjs}/<name>.js; synced SDK at
        │                                     dist/{esm,cjs}/src/**.)
        │  npm run build:smoke                (verifies ESM + CJS both expose the 12
        │                                     expected names and all 5 CJS subpaths
        │                                     resolve; wired into prepublishOnly.)
        ▼
wrapper/dist/**  (the publishable artefact)
        │
        │  npm pack --dry-run                (verify the tarball; 5812 files,
        │                                     525.4 kB as of v0.1.0 [Unreleased].
        │                                     Roughly doubled vs ESM-only baseline
        │                                     because every file emits twice — once
        │                                     ESM, once CJS — plus per-format .d.ts.)
        ▼
clockify-sdk-ts@<version>.tgz  →  npm publish via release.yml on v*.*.* tag
```

`fern generate` runs the SDK template inside Docker
(`fernapi/fern-typescript-node-sdk:3.71.2`). Local Docker daemon must
be running. CI runs the same container.

## 4. Verify gates (run BEFORE every commit that changes any of:
spec sources, the generator, wrapper code, workflows, package.json)

| Change scope              | Run                                                              |
|---------------------------|------------------------------------------------------------------|
| Generator (`../GOCLMCP/scripts/gen-clockify-openapi`) | `make gen-openapi` + all 4 drift gates + `go test ./internal/tools/...` |
| Upstream sources (GOCLMCP/docs/openapi/sources)    | same as generator                                  |
| `spec/corrected/` snapshot only                    | should never happen — see §5                       |
| `spec/fern/generators.yml` / `fern.config.json`    | `fern check --warnings --from-openapi` + `fern generate --group ts --local --force` |
| `wrapper/src/**`                                   | not allowed — wiped by `npm run sync`              |
| `wrapper/scripts/sync-sdk.sh`                      | run `npm run sync` and verify file count is sensible (currently 723) |
| `wrapper/{index.ts, create-client.ts, composed-fetch.ts, iter.ts, webhooks.ts, pagination.ts}` (hand-written modules) | `npm run type-check` + `npm test` (unit cases live in `tests/<module>.test.ts`) + `npm run build` + `npm run build:smoke` + `npm pack --dry-run`. After adding a new hand-written module, also add it to `tsconfig.json` `include`, `tsconfig.esm.json` `include`, `tsconfig.cjs.json` `include`, a subpath entry in `package.json` `exports` (both `import` and `require` conditions), and the expected-names array in `scripts/verify-dual-build.sh`. |
| `wrapper/CHANGELOG.md`                              | edit-only, no gates — runs alongside the package metadata changes that prompted the entry |
| `wrapper/{package.json, tsconfig*.json, README.md, LICENSE, vitest.config.ts, tests/**, examples/**}` | `npm run type-check` + `npm test` + `npm pack --dry-run`. Examples are type-checked via `tsconfig.json`'s `include` even though they're not in any test/build output — drift in the synced SDK that breaks an example signature fails the type-check. |
| `.github/workflows/**`                             | the security-guidance hook may block the first Write per session; retry once; lint with `gh workflow view <name>` |
| `wrapper/typedoc.json`                              | `npm run docs` (regenerates `docs/api/`; failures here block the Phase 3.3 docs workflow). |

After any spec or generator change, the FULL chain in §3 must run
end-to-end and end green before push. Drift gates are non-negotiable.

## 5. Critical conventions (the rules that bite)

1. **Never edit `spec/corrected/clockify.corrected.openapi.yaml`
   directly.** It is a regenerable snapshot. Edits land in the
   upstream sources (`GOCLMCP/docs/openapi/sources/**`) or in the
   generator script. If `make gen-openapi` would overwrite your
   change, your change is in the wrong place.
2. **Never edit `output/ts-sdk/**` directly.** `fern generate
   --force` wipes that tree on every regen. Hand-written code lives
   in `wrapper/`, not in `output/`.
3. **Never edit `wrapper/src/**` directly.** `npm run sync` wipes
   and repopulates it from `output/ts-sdk/`. The sync script
   intentionally skips `package.json`, `tsconfig*.json`,
   `node_modules/`, lockfiles, and `.gitignore` so the wrapper's own
   versions of those files survive.
4. **Never commit the raw probe files
   (`spec/evidence/probes/*.{json,hdr}`).** They are git-ignored
   already. Promote canonical findings into
   `spec/evidence/discrepancies.md` and reference the probe by
   relative path.
5. **Never run `npm publish` from a developer laptop without a
   `--dry-run` first.** The release workflow exists for a reason.
6. **Never push a tag that doesn't match `package.json` version.**
   The release workflow's tag-vs-version guard will fail the job;
   the consequence is a half-burnt git tag that needs cleanup.
7. **Drift gates must pass after every spec change.** All four:
   `openapi-drift`, `catalog-drift`, `selfinspect-drift`,
   `raw-allowlist-drift`. Skipping any of them is silent
   data corruption — the GOCLMCP MCP layer derives its tool catalog
   from the canonical spec; drift here breaks the MCP downstream.
8. **`go test ./internal/tools/...` in GOCLMCP must pass after every
   spec change.** Same reason as above — the Go tool layer parses
   the canonical YAML; a malformed change breaks the MCP server.
9. **Do not introduce `it.skip` / `test.skip` / `xit` / `xdescribe`
   in `wrapper/tests/`.** Use the env-gated `describe.skip` pattern
   already in `tests/sandbox.test.ts` for live tests, but never
   skip something silently.

`"files": ["dist", "README.md", "LICENSE"]` in `package.json`
whitelists what `npm publish` ships. Do not add to that list without
a publish-readiness review. `CHANGELOG.md` is intentionally omitted
to keep the tarball lean.

The package exposes six subpaths via `package.json` `exports`,
each with `import` + `require` conditions (modern dual-tier shape:
`{ types, default }` per condition so TS resolves ESM vs CJS types
correctly):
- `clockify-sdk-ts` → `./dist/{esm,cjs}/index.js` (package root —
  re-exports the synced SDK surface + `createClockifyClient` +
  `iterAll` / `iterPages` / `KNOWN_PAGINATED_METHODS` + `paginate` +
  `verifyClockifyWebhook` / `constructEvent` / `WebhookSignatureMismatchError` +
  `composedFetch` / `defaultUserAgent` / `generateRequestId` /
  `getRequestIdFromError` and all related types).
- `clockify-sdk-ts/create-client` → `./dist/{esm,cjs}/create-client.js`
  (the `createClockifyClient()` factory in isolation).
- `clockify-sdk-ts/composed-fetch` → `./dist/{esm,cjs}/composed-fetch.js`
  (the standalone fetch wrapper — UA / req-id / hooks / retry policy).
- `clockify-sdk-ts/iter` → `./dist/{esm,cjs}/iter.js` (per-resource
  pagination helpers + `KnownPaginatedMethod` documentary union).
- `clockify-sdk-ts/webhooks` → `./dist/{esm,cjs}/webhooks.js` (Clockify
  webhook signature verifier + JSON-payload parser).
- `clockify-sdk-ts/pagination` → `./dist/{esm,cjs}/pagination.js`
  (low-level callback-style `paginate<T>` helper).

`package.json` also carries `publishConfig: { access: public,
provenance: true }` — `npm publish` from any environment publishes
publicly with sigstore provenance by default. The
`release.yml` `--access public --provenance` CLI flags are now
redundant; kept for defense-in-depth but no longer load-bearing.

The `addonToken: (() => undefined) as unknown as () => string`
cast is a known workaround for a Fern typing limitation (see
`spec/evidence/discrepancies.md` →
`fern.sdk.auth.addonToken-typed-required-but-mutually-exclusive`).
The wrapper's `createClockifyClient()` factory (at
`clockify-sdk-ts/create-client`) hides this cast behind a typed
discriminated-union API so end users don't see it. The raw cast
still lives inside the factory and inside `tests/sandbox.test.ts`
(which exercises the raw `ClockifyApiClient` constructor directly).
Remove the cast everywhere only after the upstream Fern type is
fixed.

## 7. Live tests (env-gated; sandbox-only)

`wrapper/tests/sandbox.test.ts` reads `CLOCKIFY_API_KEY` and
`CLOCKIFY_WORKSPACE_ID` from the shell. Tests skip cleanly if either
is absent (CI runs without them deliberately).

**Never run live tests against a customer workspace.** The CRUD
round-trip creates and deletes records; an accidental run against
production data would litter the workspace.

When adding new live flows:
- Always pair create with delete in the same `it` block.
- Use timestamp-prefixed slugs (`sdk-test-${Date.now()}`) so
  litter from an aborted run is identifiable.
- Set `testTimeout: 30_000` (already in `vitest.config.ts`); Clockify
  is occasionally slow on first-page requests against large lists.
- Treat any 401 / 5xx as a test bug, not a spec bug, until proven
  otherwise. Run the curl equivalent by hand to confirm the live API
  shape before changing assertions.

## 8. Known deferred / blocked items

Tracked in `spec/evidence/discrepancies.md` with full repro evidence:

1. `fern.x-fern-pagination.bare-array-unsupported` — Fern CLI 5.37.9
   rejects `results: $response` for bare-array responses; Clockify
   has no envelope. **The SDK does not auto-paginate via Fern.**
   The wrapper ships a hand-written `paginate<T>` helper (see §6) as
   the supported workaround: `import { paginate } from
   "clockify-sdk-ts/pagination"`. Manual `page` / `page-size` loops
   also work (`wrapper/README.md` shows both forms). Re-evaluate
   Fern's bare-array support on each upstream CLI bump.
2. `fern.x-fern-sdk-method-name.drops-resource-modules` — a CRUDL
   heuristic on `x-fern-sdk-method-name` caused Fern to silently drop
   12 of 31 resource modules from TS output. Heuristic parked as a
   NOTE in the generator; SDK method names remain operationId-derived
   (long but stable: `client.tags.getWorkspacesWorkspaceIdTags(...)`).
3. `fern.sdk.auth.addonToken-typed-required-but-mutually-exclusive`
   — Fern types both `apiKey` and `addonToken` as required; Clockify
   accepts exactly one. Workaround documented; needs upstream fix.

**Recently resolved (kept here as a pointer):**
- `deferred-list-endpoints.not-paginated-or-not-live` (2026-05-24,
  session 2 re-probe). The three deferred list endpoints
  (`/workspaces` top-level, `/workspaces/{wsId}/balance`,
  `/workspaces/{wsId}/holidays/in-period`) were confirmed
  **not paginated** (or, for bare `/balance`, not live at all).
  None added to `PAGINATED_LIST_OPS`. The granular
  `time-off/balance/policy/{policyId}` and
  `.../user/{userId}` routes — already in `PAGINATED_LIST_OPS` —
  are the live equivalents.

Re-attempt items 1-3 only after the upstream gating concern is
resolved (Fern issue acknowledged or a workaround discovered).

## 9. Secret hygiene

- `CLOCKIFY_API_KEY` and `CLOCKIFY_WORKSPACE_ID` belong in the
  developer's shell env. Never commit them. Never echo them
  unredacted to a log file. Never paste them in a Slack / GitHub
  comment.
- `NPM_TOKEN` lives in the repo's GitHub Actions secrets
  (`Settings → Secrets and variables → Actions`). Use an
  **automation** token with **Publish** scope, **no expiry** or
  ≤1 year, and rotate after every push to npm.
- `gh auth status` token belongs to a single developer machine, not
  to CI. CI uses the per-job `GITHUB_TOKEN` for read-only checkout.

## 10. Commit & branch hygiene

- Conventional prefixes: `feat:`, `fix:`, `chore:`, `docs:`,
  `refactor:`, `test:`, `ci:`. One logical change per commit.
- The subject line MUST fit in 72 chars. Wrap the body at 72 chars
  too.
- Generated code (under `output/ts-sdk/`) is committed for CI's
  `npm run sync` step. When the generator change touches many files,
  describe the *change to the generator*, not the diff to the
  generated files — the diff is implied. Example commit message:
  > `feat(gen): stamp page+page-size on 18 list endpoints`
- Never push to `main` from a feature branch via PR-merge UI without
  CI green on the PR head. Direct pushes to `main` are reserved for
  hotfixes you can defend in writing.
- Do not amend a published commit (`git commit --amend` followed by
  `git push --force`). Add a new commit on top.
- Do not skip hooks (`--no-verify`) or bypass signing
  (`--no-gpg-sign`). If a hook fails, fix the underlying issue.

## 11. Doc maintenance

- Every spec-shape change ships with a corresponding entry in
  `spec/evidence/discrepancies.md` using the five-question format.
  An entry is not a substitute for fixing the issue; it is a
  *trail* that lets the next agent (human or AI) understand why
  the code looks the way it does.
- `wrapper/README.md` is the SDK's npm-facing README. Update it
  for any user-visible change: new method, new auth requirement,
  changed pagination shape, deprecation. Do not duplicate internal
  build-chain detail there; that lives in this `AGENTS.md` and the
  workspace `README.md`.
- `wrapper/CHANGELOG.md` exists (added 2026-05-24, session 2) in
  Keep-a-Changelog format with an `[Unreleased]` heading on top.
  Every user-visible change between releases lands under
  `[Unreleased]`; on tag day, that section is renamed to the new
  version + ISO date. Each version entry references the relevant
  `discrepancies.md` anchors for any limitation it inherits, ships,
  or closes. `CHANGELOG.md` is intentionally not in
  `package.json` `files`, so the npm tarball stays lean; readers
  follow the repo URL from the manifest.

## 12. Out of scope (FLAG and stop)

These changes require explicit author approval before any code or
config touches them. If asked to do one of these without prior
buy-in, surface the request and stop:

1. Editing `../GOCLMCP/scripts/gen-clockify-openapi`'s
   merge / dedup logic (lines 700–880 region). One bug there
   destroyed 26 request-body interfaces last cycle. Add to
   `PATH_PARAM_PATTERNS`, `PAGINATED_LIST_OPS`, or `TAG_RENAMES`
   freely — those are data-only.
2. Renaming the npm package (`clockify-sdk-ts`). The repo's git
   name (`clockify-ts-sdk`) and the npm name diverged intentionally.
3. Changing the Fern CLI version (`spec/fern/fern.config.json`'s
   `version: "5.37.9"`) or generator container version
   (`spec/fern/generators.yml`'s
   `fernapi/fern-typescript-node-sdk:3.71.2`). Bumps require a full
   regression cycle.
4. Removing the `addonToken` workaround cast in `wrapper/`. It
   pretends to satisfy Fern's typing while sending the right
   header at runtime — removing it without an upstream Fern fix
   breaks auth.
